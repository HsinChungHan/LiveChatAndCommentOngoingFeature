# 變更日誌

本目錄用於記錄每日的 code change 和新增內容，確保開發過程的變更都有記錄。

## 文件說明

- `CHANGELOG.md`：主要的變更日誌，按日期記錄所有變更

## 使用方式

1. **每次 commit 後**：記錄相關的變更
2. **每天結束前**：使用 Cursor 協助生成當天的變更記錄
3. **定期回顧**：檢查變更是否符合 TDD 規劃

## 變更類型

- **✅ 完成**：新功能或 Ticket 完成
- **🚧 進行中**：正在進行的功能
- **🐛 Bug 修復**：修復的問題
- **♻️ 重構**：程式碼重構
- **📝 文件更新**：TDD 或文件更新
- **⚠️ 問題發現**：發現的問題或風險

## 與 Cursor 協作

每天結束時，可以這樣與 Cursor 互動：

```
根據今天的 git commits，幫我生成今天的變更記錄：
1. 列出所有修改的檔案
2. 對應到相關的 TDD ticket
3. 生成 CHANGELOG.md 的格式
4. 更新 implementation_status.md
```
