# 工作日誌

本目錄用於記錄每日的開發工作，包含計劃、完成項目、遇到的問題等。

## 文件說明

- 每日一個文件，命名格式：`YYYY-MM-DD.md`
- 例如：`2025-01-15.md`

## 使用方式

1. **每天開始時**：建立當天的工作日誌文件，列出今日計劃
2. **實作過程中**：隨時記錄進度、問題、學習心得
3. **每天結束時**：完成當天的工作日誌，並生成明天的計劃

## 與 Cursor 協作

### 每天開始時
```
今天要實作 TDD-001 和 TDD-002，請：
1. 讀取相關的 TDD 文件
2. 檢查現有的 codebase 結構
3. 提供實作建議和注意事項
4. 生成工作日誌模板
```

### 每天結束時
```
請幫我：
1. 根據今天的實作更新工作日誌
2. 更新 implementation_status.md
3. 更新 CHANGELOG.md
4. 檢查 TDD 與實作的一致性
5. 生成明天的計劃建議
```

## 工作日誌模板

每個工作日誌文件應包含以下部分：

1. **📋 今日計劃**：列出今天要完成的工作
2. **✅ 完成項目**：記錄完成的工作
3. **🚧 進行中項目**：記錄進行中的工作
4. **🐛 遇到的問題**：記錄遇到的問題和解決方案
5. **🤖 與 Cursor 的對話統整**：記錄重要決策、學習要點、問題解決記錄（**必須包含 User Prompt 欄位**）
6. **💡 學習與反思**：記錄學習心得和反思
7. **📊 時間統計**：記錄時間分配
8. **🔄 明日計劃**：列出明天的工作計劃
9. **📄 相關文件改動記錄**：記錄 Git commits 和文件統計

## 格式規範

**重要**: 所有工作日誌必須遵循標準格式規範，請參考：

- [格式規則](./FORMAT_RULES.md) - 完整的格式規範和範例
- [模板文件](./TEMPLATE.md) - 工作日誌模板
- [工作流程](./WORKFLOW.md) - Git 分支工作流程

### 關鍵格式要求

1. **完成項目格式**: 使用 `### TDD-XXX — [標題] [JIRA-ID]` 格式，項目之間用 `---` 分隔
2. **User Prompt 欄位**: 在「🤖 與 Cursor 的對話統整」中，每個決策和問題解決記錄都必須包含 `User Prompt` 欄位
3. **時間格式**: 使用 `~X 分鐘` 或 `~X 小時`
4. **檔案路徑**: 使用反引號包裹，例如 `` `路徑/檔案名.swift` ``
